## ⚡ STEP-BY-STEP PROCESS

### STEP 0 — Do this before your hackathon starts (30 min)
1. One person (Karan recommended) creates a **GitHub repo** named `disease-ews`
2. Add all 4 members as collaborators
3. Everyone clones it: `git clone https://github.com/your-org/disease-ews.git`
4. Upload the base Phase 1 files to the repo (from `prompts/PHASE1_BASE_FILES/`)

### STEP 1 — Everyone runs Phase 1 (20 min, do together)
- Follow `prompts/EVERYONE_PHASE1.txt`
- At the end: `python backend/init_db.py` should print ✅ 22 districts seeded
- Flask at `localhost:5000/api/health` should return `{"status":"ok"}`

### STEP 2 — Split and work in parallel (30–35 hrs)
- Arpit opens `prompts/ARPIT_PHASE2_DATA.txt` → paste into Claude → follow instructions
- Karan opens `prompts/KARAN_PHASE3_4_ML.txt` → paste into Claude → follow instructions
- Vishal opens `prompts/VISHAL_PHASE5_API.txt` → paste into Claude → follow instructions
- Tanya opens `prompts/TANYA_PHASE6_FRONTEND.txt` → paste into Claude → follow instructions

### STEP 3 — Integration (6–8 hrs before deadline)
- Follow `INTEGRATION.md` exactly
- One person (Karan) does the merge on his laptop
- Everyone pushes their files to GitHub first

### STEP 4 — Final demo prep (2 hrs)
- Load demo data: `python backend/ingest.py --demo`
- Run full app: backend + frontend together
- Test the "Simulate Outbreak" button
- Practice the presentation script
